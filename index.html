编程语言中的命名规则

一、驼峰明明法
小驼峰命名法
格式：第一个单词首字母小写，后续单词首字母大写，无下划线。
eg:lowerCamelCase
适用范围：适用场景：绝大多数编程语言中的变量、函数 / 方法、对象属性

大驼峰命名法
格式：每个单词的首字母都大写，无下划线，也叫帕斯卡命名法。
eg:UpperCamelCase / PascalCase
适用范围：适用场景：类、接口、结构体、枚举、全局常量（部分语言），是面向对象编程中的核心命名规范。

二、 常量命名法
这是下划线命名法的变种，也叫全大写下划线命名法，是常量的专属命名规范。
格式：所有字母大写 + 单词之间用 _ 连接。
eg:const MAX_RETRY_COUNT = 3;
适用范围：适用场景：所有编程语言中的常量（包括全局常量、枚举值、配置项中的固定值）。

三、 下划线命名法
下划线命名法的特点是所有字母小写，单词之间用下划线 _ 分隔，风格简洁直观，不易出错。
eg:columns: user_id, user_name, create_time，system_init;
适用范围：函数名，复合变量名。

四、 连字符命名法
连字符命名法的特点是所有字母小写，单词之间用连字符 - 分隔，也叫短横线命名法。
格式：全小写 + 单词之间用 - 连接。
eg:user-info-title {
font-size: 18px;
}

单片机寄存器

锁存器（latch）—对脉冲电平敏感，在时钟脉冲的电平作用下改变状态
触发器(flip-flop)—对脉冲边沿敏感，其状态只在时钟脉冲的上升沿或下降沿的瞬间改变
寄存器（register）：用来存放数据的一些小型存储区域，用来暂时存放参与运算的数据和运算结果

16位寄存器
1111 1111 1111 1111
GPIOC->ODR&=0x00FF：清除GPIOC数据寄存器高8位，保留低八位
GPIOC->ODR|=0xFF<<8;GPIOC高8位的数据寄存器设为高电平

BSRR：
STM32的BSRR是32位寄存器，按功能分为两个16位区域（对应GPIO的16个引脚Px0 ~ Px15），仅支持写操作，读操作返回无效值。
BRR：
就是BSRR的高16位置零寄存器

第 0 ~ 15位 |BS（Bit Set） |置 1 区域：写 1 对应引脚输出高电平，写 0 无效果| 第8位写1→PC8 置高 |

第 16 ~ 31位|BR（Bit Reset）|置 0 区域：写 1 对应引脚输出低电平，写 0 无效果| 第24位写1→PC8 置低|

eg:GPIOC->BSRR=0x01<<8; //PC8置高电平
GPIOC->BSRR=0x01<<16+8; //PC8置低电平 <==>GPIOC->BRR=0x01<<8;

关键规则：
无论是BS区域还是BR区域，只有写1有效，写0完全不影响对应引脚。
若同时对某一引脚的BS位和BR位写 1，BS位（置 1）优先级更高。

计算机硬件基础


缓存：位于CPU内部，（L1、L2、L3）存储速度最快，协助CPU处理数据
内存：也叫memory，位于CPU外部，存储速度次之，协助CPU处理数据
硬盘：长期存储系统，软件，文件（断电不丢失）

RAM：缓存SRAM(静态RAM),内存DRAM（动态RAM，充放电擦除所以慢一点）

CubeMX结合keil使用的基础配置

CubeMX结合Keil使用的基础配置
时钟配置：
input frequency–晶振的频率24Mhz(蓝桥杯嵌入式板子) HSE-PLLCLK-APB1、APB2根据题目设置
SYS配置：
蓝桥杯的不用设置，其余情况根据下载器的协议选择
NVIC配置：
System tick timer 优先级设置为0
工程的配置：
固件包选择GW1.4.0版本
keil的配置：
菱形取消勾选；target/微库√；C/C++/优化等级设0； Debug/setting/trace-flash download

.mxproject文件包含项目的元数据和配置
core目录包含核心代码：主程序、系统初始化（时钟、外设）、中断服务例程
Driver目录包含设备驱动程序，通常包含外设的初始化和控制代码（GPIO.UART.I2C等外设的控制接口），设备抽象层（HAL/LL库）
MDK-ARM目录用于存放ARM MDK集成开发环境的相关环境
user存放用户自定义的代码目录
.ioc文件是一个I/O配置文件，通常用于描述控制器的引脚和外设配置

新建system.c/.h
system.c：
#include “system.h” （ #include <>：用于引入系统 / 标准库头文件；#include “”：用于引入用户自定义头件）
system.h的预编译：
#ifndef __SYSTEM_H
#define __SYSTEM_H

#endif

新建bsp_system.h
bsp_system.h存放该工程所有的公共变量，结构体，枚举等；以及所有需要链接的到的库以及所有.h文件；
各个子文件中只需要包含#include “bsp_system.h”
