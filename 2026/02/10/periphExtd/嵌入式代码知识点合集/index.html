sprintf的作用
sprintf是 C 语言标准库（<stdio.h>）中的格式化字符串函数
核心功能是把变量（int/float/char 等），按指定格式转换成字符串，并写入到字符数组（缓冲区）中。也叫做拼接变量。
例如： sprintf(buf, “ADC值：%d，电压：%.2fV”, adc_val, volt);
// 此时buf中的内容是：”ADC值：2048，电压：1.65V”

UART_TypeDef和UART_HandleType的区别
UART_TypeDef：UART 外设的寄存器映射结构体
UART_TypeDef 是 STM32 标准库 / HAL 库中定义的寄存器地址映射结构体，每一个成员对应 UART 外设的一个物理寄存器（如 SR、DR、BRR 等），其首地址直接对应单片机内存中 UART 外设的硬件寄存器地址。
例如：
typedef struct {
__IO uint32_t SR; // 状态寄存器
__IO uint32_t DR; // 数据寄存器
__IO uint32_t BRR; // 波特率寄存器
__IO uint32_t CR1; // 控制寄存器1
// … 其他寄存器（CR2、CR3、GTPR等）
} UART_TypeDef;
// 预定义：将结构体映射到实际硬件寄存器地址（如USART1基地址0x40013800）
#define USART1 ((UART_TypeDef *)0x40013800U)
#define USART2 ((UART_TypeDef *)0x40004400U)//USART1、USART2是指向某特定串口硬件的地址

UART_HandleTypeDef：HAL 库的 UART 操作句柄
UART_HandleTypeDef包含了指向 UART_TypeDef 的指针（Instance），还封装了 UART 的配置参数、运行状态、回调函数等。
标准定义（HAL 库 stm32g4xx_hal_uart.h 中）：
typedef struct __UART_HandleTypeDef {
UART_TypeDef* Instance; // 指向UART寄存器结构体的指针（核心关联）
UART_InitTypeDef Init; // UART配置参数（波特率、数据位、停止位等）
uint8_t* pTxBuffPtr; // 发送缓冲区指针
uint16_t TxXferSize; // 发送数据长度
…
} UART_HandleTypeDef;
其中huart1.Instance=USART1;

uwTick介绍
uwTick=HAL_IncTick(),即从系统启动到现在的时间（ms）；
基本单位：默认 1 毫秒（ms）= 1000 微秒（μs）
数据类型：uint32_t（32 位无符号整数），最大值为 0xFFFFFFFF（约 49.7 天），溢出后会从 0 重新计数

int变量和char变量的本质区别：
当变量值用于字符类输入输出传输场景时，系统硬件会默认将其解析为 ASCII 码；此时不论是char还是int类型都存储与传输的都是ASCLL码值。

pRxBuffPtr是UART接收数据的缓冲区指针，HAL库在接收中断中，会把接收到的每一个字节依次存入该指针指向的内存地址，每存一个字节，该指针会自动向后偏移1位。

串口调试助手模式组合对照表
接收：ASCII 接收数字解析显示为ASCLL码对应字符 eg:发送0x41 串口助手接收为’A’;
发送：ASCII 发送字符解析并发送为ASCLL码值 eg:串口助手发送’A’ 电脑接收到0x41
接收：Hex接收数字解析显示为ASCLL码值 eg:发送’A’ 串口助手接收为0x41;
发送：Hex发送数字解析并发送为对应ASCLL码对应的字符eg:串口助手发送41 电脑接收到’A’;
